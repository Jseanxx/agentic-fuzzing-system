# Final Goal Update — 2026-04-14

## 최종 목표
최종적으로는 Hermes를 이용해 **Proxmox 쪽 퍼징 머신/VM에 SSH 접속해서**, 원격에서도 하네스 수정 → 시드 조정 → 퍼징 재실행 → 로그 분석 → 기록 업데이트가 이어지는 **자가발전형 퍼징 에이전트 운영 체계**를 만드는 것이다.

## 단계적 목표
### Stage 1 — WSL 로컬 검증
- 지금 쓰는 노트북 WSL2에서 build / smoke / fuzz / 기록 / 정책 분기가 안정적으로 도는지 검증

### Stage 2 — 반자동 정책화
- triage / coverage / regression 모드 분리
- corpus / artifact / checklist / 정책 문서화
- 수정 후 재검증 루프를 강제

### Stage 3 — 원격 이식 준비
- 로컬에서 검증된 실행 흐름을 SSH 대상에서도 재현 가능하게 스크립트화
- 원격 artifact 회수/요약/보고 체계 정리

### Stage 4 — Proxmox 원격 퍼징 운영
- Hermes가 SSH로 접속
- 원격 빌드, 퍼징 실행, 결과 수집, 기록 반영
- 필요 시 하네스/시드/정책 재조정

### Stage 5 — GitHub 공개
- 퍼징 자동화 시스템의 구조/정책/프롬프트/한계/데모를 정리해서 공개
- 개인 환경 의존 부분과 일반화 가능한 부분을 분리 설명

## 중요한 원칙
- 지금 단계에서 가장 중요한 것은 ‘멋진 자동화’가 아니라 **검증 가능한 운영 루프**다.
- 성공 판정은 단발성 크래시가 아니라, 피드백을 보고 다음 행동이 이어지는지 여부다.
